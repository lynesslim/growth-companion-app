# Deepseek Implementation Plan - Phase 3

## Goal
Implement Phase 3 of the feedback: Social & Streak Logic Adjustments.

## Objectives
1. **Post-Reading Modal (Item #2):** When a user completes their daily book (`StreakCompleteScreen` or similar), and they navigate back to the Home Page, trigger a modal popup showing "Check out your friends' streaks and gifts!" This should ONLY trigger ONCE per day.
2. **Share Text Update (Item #5):** Update all "Share" button texts across the app (in `StreakCompleteScreen`, `FriendsScreen`, `SocialScreen`) to use this exact template: "Look what I'm learning on Growth Companion: [Link]". Make sure you use the `share_plus` package properly (e.g. `Share.share(...)`) and replace any deprecated `share()` calls you see.
3. **24h Streak Expiration (Item #7):** Ensure that if a user does not read a book within 24 hours, their streak automatically expires (resets to 0). You will need to implement this check in the relevant domain model/provider logic (e.g., `UserProvider` or `SocialProvider`) when the app calculates the user's current streak.

## Instructions & Ponytail Constraints
- **YAGNI:** For the Post-Reading Modal, just track the "last shown date" in `SharedPreferences`. Do not build a complex database tracking system for a simple daily modal.
- Use `Share.share(text)` from `share_plus` correctly to avoid the deprecated `Share` API warnings you saw in the analyzer logs earlier.
- Ensure the Streak expiration check is robust and gracefully handles timezone issues by strictly comparing `DateTime.now().difference(lastReadDate).inHours > 24`.
