Please implement the following exact changes to fix the invite link flow and modal race conditions.

### 1. Fix Splash Screen blocking `/invite` route
- **File**: `lib/src/core/router.dart`
- **Action**: Locate the `// Enforce splash screen on first boot` redirect block.
- **Change**: Exempt the `/invite` route so that the `InviteLandingScreen` can properly load and stash the invite tokens before the user is forced through the onboarding funnel.
```dart
      // Enforce splash screen on first boot
      if (!SplashState.hasFinished && state.matchedLocation != '/splash' && state.matchedLocation != '/invite') {
        return '/splash';
      }
```

### 2. Fix "Today's Drop" modal popping up during onboarding
- **File**: `lib/src/features/auth/login_screen.dart`
- **Action**: Completely delete the `ref.listenManual(authStateProvider)` listener block inside of the `initState()` method.
- **Reason**: `LoginScreen` currently forces a premature `context.go('/')` upon signup/login. This races against the native `GoRouter` redirect, causing the `HomeScreen` to momentarily instantiate and fire its modal popups *underneath* the onboarding screen. By deleting this listener, we allow GoRouter's global `redirect` block to cleanly handle the routing natively.
